本文档用于记录当前项目已确认、但暂不立即处理的优化项，避免后续重复讨论。

## 1. 降低入口重复

目标：减少 `README.md`、`AGENTS.md`、`llms.txt` 之间的重复信息，降低维护成本。

建议分工：

- `README.md`：给人类读者看，说明项目是什么、适合谁、从哪里开始读
- `AGENTS.md`：只放 Agent 开发规则和仓库使用约定
- `llms.txt`：只保留给 LLM 的压缩索引和快速判断信息

当前状态：

- 已处理：根目录 `AGENTS.md` 作为当前知识库仓库唯一稳定主规则入口，`llms.txt` 只做索引和导航。
- 已处理：`README.md`、根目录 `AGENTS.md`、`llms.txt` 的入口顺序已统一为“先遵循 `AGENTS.md`，再用 `llms.txt` / 子目录索引定位文档”。
- 已处理：`xbot-api-docs/AGENTS.md` 已压缩为防漏规则，不再完整重复根目录规则。
- 已处理：`xbot-api-docs/README.md` 和 `xbot-api-docs/docs/README.md` 作为人类阅读入口已删除，Agent 通过 `llms.txt` 和 `xbot-api-docs/llms.txt` 定位文档。

## 2. 核心文档补来源标记

目标：让 Agent 或开发者能快速回溯关键结论对应的源码来源，减少猜测。

当前共识：

- 不能只写模糊来源
- 关键结论应尽量保留源码全路径定位能力
- 优先使用“小节级来源块”，避免正文过重
- 市场指令类内容除了源码路径，最好同时标注调用入口类型，如 `flow` / `direct python`

建议适用范围：

- `xbot-api-docs/docs/` 下的核心 API 文档
- `xbot-api-docs/docs/extension-instructions.md`
- `xbot-api-docs/docs/debug/market-extension-source.md`

当前状态：

- 先记为待办，等后续真正需要补时再统一处理格式
