# 价格监控方案

## 推荐分工

- **影刀 / Python 自动化**：负责打开网页、登录、读取价格、写回表格
- **OpenClaw / Agent**：负责定时调度、异常判断、汇总通知、日报

## 为什么不建议让 Agent 直接自由判断页面价格

价格监控的核心是**稳定取数**，不是自由推理。页面上可能存在多种价格：

- 券后价
- 活动价
- 到手价
- 广告价

应优先使用以下方式取数：

- 固定 XPath / CSS 选择器
- 页面接口（XHR / API 响应）
- 明确规则（如取第一个 `.price` 文本）

## 建议的数据表结构

| 字段 | 说明 |
|---|---|
| 商品名称 | 商品标识 |
| 平台 | 淘宝、京东、拼多多等 |
| 商品链接 | 商品页面 URL |
| 目标价 | 期望价格（用于降价提醒） |
| 当前价 | 最新采集价格 |
| 上次价格 | 上一次采集价格（用于判断涨跌） |
| 采集时间 | 本次采集时间戳 |
| 状态 | 见下表 |
| 备注 | 异常原因、通知记录等 |

## 状态值

| 状态 | 含义 |
|---|---|
| 正常 | 价格采集成功，无异常 |
| 降价 | 当前价低于上次价格 |
| 涨价 | 当前价高于上次价格 |
| 采集失败 | 页面结构变化、网络异常等 |
| 需要登录 | 登录态过期或需要验证码 |
| 页面异常 | 页面返回 404、被拦截等 |

## 关键步骤

1. **读取商品列表**：从 Excel 或数据库读取待监控商品
2. **打开商品页面**：使用 `xbot.web` 打开链接（参考 `docs/browser.md`）
3. **提取价格**：使用元素查找获取价格文本
4. **对比判断**：与目标价、上次价格对比，更新状态
5. **写回表格**：使用 `xbot.excel` 更新结果（参考 `docs/excel.md`）
6. **通知汇总**：Agent 读取结果，生成日报或异常告警

## 注意事项

- 频繁访问同一网站可能触发反爬，建议设置合理间隔
- 登录态需要定期刷新，可单独维护 Cookie 或扫码登录流程
- 价格文本可能包含 "¥"、"," 等符号，采集后需清洗为纯数字
- 大量商品监控时，优先考虑后台 API 请求，减少浏览器打开次数
