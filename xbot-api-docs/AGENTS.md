# AGENTS.md

本仓库是给 AI Agent 使用的影刀 ShadowBot / xbot API 开发指南。`xbot-api-docs` 是独立仓库目录，Codex 等工具如果在该目录启动，可能只读取这里的 `AGENTS.md`，因此本文件需要保留核心规则。

## 使用原则

1. 写影刀编码版代码前，优先查 `docs/`；如果已知有对应的 `llms.txt`，先读 `llms.txt` 再读具体文档。
2. 浏览器操作查 `docs/browser.md`。
3. 表格操作查 `docs/excel.md`。
4. 键盘鼠标操作查 `docs/keyboard-mouse.md`。
5. 市场指令参数不明确时，查 `docs/debug/market-extension-source.md`。
6. 浏览器、URL、网页业务功能默认只允许使用 `xbot.web`；除非用户明确要求，不要使用 `requests`、`httpx`、`aiohttp`、`urllib` 或其他网络相关 Python 库。
7. 不要根据界面中文选项猜编码版参数。
8. 不确定 API 行为时，标注“需运行验证”，不要写成确定结论。
9. 任务涉及影刀 API、xbot 指令、开发流程或文档维护时，Agent 应主动读取对应文档；`llms.txt` 不是所有 Agent 都会自动读取。
10. 优先使用简单、直接、低维护的实现。
11. 不要过度封装，不要为了小任务引入复杂抽象。
12. 能用元素操作就不要优先坐标点击。
13. 中文、长文本输入优先考虑剪贴板输入。
14. 大量表格写入优先考虑批量写入二维数组。
15. 不要提交影刀运行包源码、反编译源码、敏感路径、账号或密钥。
16. 不要改动与当前任务无关的文件、格式、命名或结构。

## 市场指令排查规则

当市场指令文档不清楚、参数不明确、可视化能跑但编码版异常时：

1. 先用 `inspect.signature()` 查看包装层参数。
2. 再用 `inspect.getfile()` 定位 `xbot_extensions` 安装目录。
3. 查看 `__init__.py`、`_core.py` 或其他真实实现文件。
4. 以源码里的真实判断逻辑为准。
5. 不要猜中文参数值。
6. 若源码判断 `close_type == "click"`，编码版就应传 `"click"`，不是 `"点击"`。

## 文档维护规则

1. 新 API 文档放到 `docs/`。
2. 可运行代码片段放到 `examples/`。
3. 每篇文档尽量包含：
   - 适用场景
   - 常用方法
   - 最小示例
   - 注意事项
   - 排错速查
4. 未验证内容必须标注“需运行验证”。
5. 发现文档与项目实际情况存在差异时，优先记录问题并在任务完成后统一汇总给用户，不要中断当前工作流去直接改文档。
